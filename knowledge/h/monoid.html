Semigroup, Monoid

Полугруппы (Semigroup) представлены классом типов, определенным в Data.Semigroup.

Semigroup определяет функцию <>: (<>) :: Semigroup a => a -> a -> a Эта функция позволяет скомбинировать две сущности для получения сущности того же типа.
Пример реализации: instance Semigroup Integer where   (<>) x y = x + y
Функция <> должна обладать ассоциативностью: неважно, в каком порядке она применяется, результат должен быть один: (a <> b) <> c = a <> (b <> c)

Monoid, как и Semigroup, определяет ассоциативную операцию <>, но, в отличие от Semigroup, требует, также, наличия нейтрального элемента: x <> id = id <> x = x id - от identity

Определение Monoid

class Monoid a where   mempty :: a   mappend :: a -> a -> a   mconcat :: [a] -> a Для списков mappend = ++ = <>

Законы Monoid

  1. Наличие нейтрального элемента слева (Left identity) mappend mempty x = x
  2. Наличие нейтрального элемента справа (Right identity) mappend x mempty = x
  3. Ассоциативность (Associativity) mappend x (mappend y z) = mappend (mappend x y) z
  4. Конкатенация (Concatenation) mconcat = foldr mappend mempty
foldr используется из-за возможности работы с бесконечными списками
