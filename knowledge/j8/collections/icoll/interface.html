Интерфейс Collection

Основополагающий интерфейс для классов коллекций. public interface Collection<E> {     boolean add(E element);     Iterator<E> iterator(); ... } Коллекции не допускают дублирование элементов

Итераторы

Интерфейс Iterator: public interface Iterator<E> {     E next();     boolean hasNext();     void remove(); } При вызове next() в конце коллекции будет выброшено NoSuchElementException.
Компилятор преобразует циклы в стиле for each в циклы с итераторами, поэтому цикл в стиле for each может использоваться с любыми объектами, реализующими интерфейс Iterable<E> (единственный метод iterator())
Порядок обхода элементов коллекции итератором зависит от типа коллекции и может быть хаотичным.
Итераторы Java представляются находящимися между элементами коллекции. При вызове next() итератор перешагивает элемент и возвращает ссылку на него. Вызов remove() удалит последний пройденный итератором элемент. Вызов remove() без предварительного вызова next() и вызов remove(), если после последнего вызова next() коллекция была изменена, выбросят IllegalStateException.
