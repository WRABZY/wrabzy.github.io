4. Неинстанцируемость

Обеспечить неиснтанцируемость можно с помощью закрытого конструктора.

Суть

В отсутствие явного конструктора автоматически создается открытый конструктор по умолчанию. Неинстанцируемости класса можно добиться, предоставив закрытый конструктор: public class UtilityClass {   private UtilityClass() {     throw new AssertionError();   }   ... }
