Стеки, деки и очереди

Стеки, деки и очереди являются ADT, то есть определяют интерфейс доступа к данным.

Стек | Stack

(Last In First Out - LIFO)

В стеке доступен только тот элемент, который был помещен в него последним. После его извлечения из стека становится доступен предпоследний и т. д.

Основные операции стека

  • Вставка элемента на вершину стека void push(T element)
  • Извлечение последнего элемента T pop()
  • Чтение последнего элемента без его извлечения T peek()
  • Проверка стека на пустоту boolean isEmpty()
  • Проверка стека на заполненность boolean isFull()

Очередь | Queue

(First In First Out - FIFO)

В очереди доступен только тот элемент, который был помещен в нее раньше остальных. После его извлечения из очереди становится доступен следующий и т. д.

Основные операции очереди

  • Вставка элемента в конец очереди void insert(T element)
  • Извлечение первого элемента в очереди T remove()
  • Чтение первого элемента в очереди без его извлечения T peek()
  • Чтение последнего элемента в очереди без его извлечения T rearPeek() Предоставляется опционально.
  • Проверка очереди на пустоту boolean isEmpty()
  • Проверка очереди на заполненность boolean isFull()

Приоритетная очередь

В приоритетной очереди элементы упорядочиваются по ключу. Новые элементы вставляются в позициях, сохраняющих порядок сортировки. Приоритетные очереди должны обеспечивать относительно быструю вставку, поэтому они часто реализуются на основе кучи.

Дек | Deque

Вставка элементов в дек и удаление элементов из дека могут производиться с обеих сторон.

Основные операции дека

  • Вставка элемента в дек слева void insertLeft(T element)
  • Вставка элемента в дек справа void insertRight(T element)
  • Извлечение крайнего левого элемента из дека T removeLeft()
  • Извлечение крайнего правого элемента из дека T removeRight()
