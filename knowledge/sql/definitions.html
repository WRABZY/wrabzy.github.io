Определения

Файл = таблица = отношение (relation)
Запись = строка = кортеж (tuple)
Поле = столбец = атрибут
Количество атрибутов называется степенью отношения, а количество кортежей - кардинальным числом.
Потенциальный ключ - это комбинация атрибутов таблицы, позволяющая уникальным образом идентифицировать строки в ней. Потенциальный ключ не должен быть избыточным. При наличии нескольких потенциальных ключей один из них используют как первичный, а остальные будут являться альтернативными ключами.
Внешний ключ - это ключ из другой таблицы. Таблица, содержащая внешний ключ, называется ссылающейся таблицей (referencing table). Таблица, (первично) содержащая потенциальный ключ, называется ссылочной (целевой, referenced table).
Ссылочное ограничение (ограничение ссылочной целостности) - значения внешних ключей должны соответствовать значениям потенциальных ключей.
Каскадное удаление (обновление) - удаление (обновление ключа) строк из ссылающихся таблиц при удалении строки (обновлении ключа) из ссылочной таблицы.
Транзакция - это набор операций над базой данных, рассматриваемых как единая и неделимая единица работы, выполняемая полностью или не выполняемая вовсе, если произошел какой-то сбой в процессе выполнения транзакции.

Операторы SQL

DDL

Data Definition Language - операторы определения данных:
  • Создание
  • Изменение
  • Удаление
таблиц, представлений и других объектов.

DML

Data Manipulation Language - операторы манипулирования данными
  • Выборка
  • Вставка
  • Обновление
  • Удаление
строк.

DCL

Data Control Language - операторы управления доступом к данным
